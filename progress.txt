## Codebase Patterns
- EnginePayloadAttributesParameter uses @JsonCreator/@JsonProperty for JSON deserialization, hex strings decoded via Bytes32.fromHexString/Bytes.fromHexString
- Test files use List.of() for non-null lists but Collections.singletonList() when nulls needed (List.of doesn't allow nulls)
- Spotless formatting required: run `./gradlew :ethereum:api:spotlessApply` before committing
- Constructor changes require updating all test call sites in AbstractEngineForkchoiceUpdatedTest and EngineForkchoiceUpdatedV2Test
- Hex string test data must have even length (after 0x prefix) for Bytes.fromHexString()
- Protocol-level EIP constants go in ethereum/core module (e.g., InclusionListConstants)
- Validation interfaces/results also go in ethereum/core (e.g., InclusionListValidator, InclusionListValidationResult)
- Use factory methods (valid(), invalid(msg), unsatisfied(msg)) for validation results with three-state status
- Engine-related CLI flags go in EngineRPCOptions/EngineRPCConfiguration (PicoCLI supports enum types natively)
- EngineRPCConfiguration is a Java record; adding fields only requires updating toDomainObject() in EngineRPCOptions
- Use getMaxGasPrice() to check if a tx can afford the base fee (returns maxFeePerGas or gasPrice); getEffectiveGasPrice returns baseFee even for underpaying txs
- TransactionTestFixture (test-support) + KeyPair for creating signed test transactions; VersionedHash first byte must be 0x01
- Error-prone UnusedMethod: unused private methods in tests cause compile failure with -Werror
- Engine API methods extend ExecutionEngineJsonRpcMethod; call engineCallListener.executionEngineCalled() at start of syncResponse()
- New Engine API methods: add to RpcMethod enum, register in ExecutionEngineJsonRpcMethods.create(), conditionally under hardfork milestone
- PendingTransaction.newPendingTransaction() requires 4 args: (tx, isLocal, hasPriority, score) - score is a byte
- AbstractEngineNewPayload.handleSuccessfulExecution() is a hook for V5+ to add post-execution validation (e.g., inclusion list)
- Use overloaded constructors for backwards compatibility when adding new parameters to factories (e.g., JsonRpcMethodsFactory, ExecutionEngineJsonRpcMethods)
